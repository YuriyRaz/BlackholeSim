# Великая Цель (Ultimate Goal)

## Цель-Абсолют

BlackholeSim — **симуляция**, где все видимые явления возникают из общих физических уравнений. Ни одно явление не должно быть "нарисовано" или "заскриптовано" — оно должно быть следствием shared physical equations.

## Способ Достижения

Строго следовать протоколу **job-orchestrator** + протоколу постановки и реализации задач через **OpenSpec**:

```
For each iteration:
1) Proposal job (openspec explore → ask Architect if issues → openspec proposal)
2) Implementation job (apply → verify → fix → sync → archive → commit & push)
3) Architect job (cross-cutting review, corrections via sub-jobs, check goals, plan next iteration)

Loop until великая цель достигнута.
```

## Непоколебимые Правила

1. **Architect смотрит всю систему**, не только свой domain. Он проверяет:
   - Не противоречит ли реализация великой цели?
   - Не накапливается ли технический долг?
   - Не основана ли новая фича на фейковом фундаменте?
   - Есть ли proposal-ы, которые блокируют текущую работу?

2. **Ни одна фича не принимается**, если она базируется на заведомо нефизическом поведении.

3. **Каждая итерация** начинается с проверки: что ещё осталось фейкового?

4. **После каждой итерации** — перечитать этот файл и скорректировать план.

5. **После сжатия контекста** — первым делом перечитать этот файл.

## Автономия принятия решений

Я действую автономно в рамках великой цели. Мои решения:

### Приоритеты

1. **Долгосрочное развитие и поддержка** — каждое решение должно облегчать, а не усложнять будущие изменения.
2. **Проект — учебное пособие для студентов-физиков** — математическая честность важнее визуальной красивости.
3. **Все ключевые математические функции физических процессов ДОЛЖНЫ быть сохранены.** Никаких "эффектов" без физической основы.
4. **НИКАКИХ костылей в физическом движке.** Формулы нельзя заменять хардкодом, таймерами, random redirect правилами или индексной логикой.

### Единственное допустимое упрощение

Формулы могут быть упрощены **только** если это необходимо для real-time рендеринга на ноутбуке без дискретного GPU. Пример: псевдо-ньютоновский потенциал вместо полной ОТО, SPH с ограниченным числом частиц.

### Критерий "достаточности" для учебного пособия

Студент-физик должен видеть:
- Как tidal force деформирует звезду (градиент потенциала)
- Как disruption происходит при пересечении радиуса приливного разрушения
- Как orbital energy и angular momentum распределяют debris
- Как формируется accretion disk через circularization
- Как GW излучение следует из ускорения масс

Всё это должно быть следствием кода, а не анимацией.
